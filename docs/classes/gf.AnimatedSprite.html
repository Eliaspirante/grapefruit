gf.AnimatedSprite

(
  • animations
  • [speed]
  • [start]
)

Defined in src/display/AnimatedSprite.js:1

Parameters:

  • animations Object

    An object of the form { animationName: [frame1, frame2] } or you can also specify overrides on a per-animation basis: { animationName: { frames: [frame1, frame2], speed: 2 } }. Each frame is a Texture object

  • [speed] Number optional

    The speed of the animations (can be overriden on a specific animations)

  • [start] String optional

    The animation to start with, defaults to the first found key otherwise

onCollision

(
  • obj
  • vec
  • colShape
  • myShape
)

Inherited from gf.PhysicsTarget but overwritten in src/physics/PhysicsTarget.js:143

On Collision Event called when this sprite collides into another, or is being collided into by another. By default if something collides with a collectable sprite we destroy the collectable and if we collide with a solid tile we kill our velocity. This method will emit a 'collision' event that you can listen for

Parameters:

  • obj Sprite

    Colliding sprite

  • vec Vector

    Collision vector (for sensors this is normalized)

  • colShape cp.Shape

    The colliding physics shape

  • myShape Sprite

    Your physics shape that caused the collision
